# Founding AI Engineer

📍 Hybrid (in-person every 2-3 months)

⏱️ Full-time

💶 $200k+ · 0.75-3%

<aside>

As a Fullstack Engineer, you’ll work at the frontier of agents, solving some of the most overdetermined problems at the intersection of AI and creative expression.

</aside>

**Stack:** Own harness based on Claude Agent SDK; Convex, Fly, Docker, TypeScript.

## What the work actually looks like:

- One week you're re-architecting the agent orchestration layer after a new model release shifts what's steerable.
- The next you're building eval tests across thousands of playable game simulations (where the output isn't abstract code, but a visible, testable game mechanic).
- In the next week, you might be analysing massive volumes of user intent data to figure out what creators mean, not just what they type (teenagers don't know how to articulate their prompts!).
- Making games used to require years of knowledge. We're teaching agents to collapse that into minutes. You're the one who technically figures out how - what context the agent sees, how it manages long-horizon tasks, and how it recovers when a 14-year-old's prompt is three words and a screenshot.

## What matters:

- You are comfortable erasing pieces of work every 3 months because the frontier moved. You've felt the whiplash of model development and you like it (you build and see instability as an edge).
- You are obsessed with evals that actually measure something - whether a game mechanic works, whether it's fun, whether a teenager stays in the creation loop. You've probably built your own benchmarks because you didn't trust anyone else's.
- You build mini tools that make repeated steps simpler. You think about token efficiency, context management, and steerability all time.
- You want freedom to do what interests you, not permission to execute someone else's spec.

## The real challenges you will face:

- Building evals at scale where the output is a playable game.
- Designing the agent's context layer so it understands user intent from noisy, incomplete prompts.
- Experimenting across agent techniques (sub-agents, orchestration patterns) and measuring which ones actually close the gap between what a kid can dream and what we can build.
- Making the whole system extensible enough that when the next SOTA drops, you can swap, benchmark on our internal evals, and ship improvements same day.

## In 3 months, you've done a great job if:

A teenager described a game mechanic with three words and a screenshot, and the system you shipped knew what they meant.
